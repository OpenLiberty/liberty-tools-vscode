# Implementation Plan: LGP 4.x Test Support with Gradle 9.0

## Overview
Add test coverage for Liberty Gradle Plugin (LGP) 4.0.0 running on Gradle 9.0 to verify compatibility and functionality.

## Background
- Current test project uses LGP 3.6.1 with Gradle 8.5
- LGP 4.0.0 requires Gradle 9.0+ (per compatibility matrix)
- Need to verify that Liberty Tools extension works correctly with LGP 4.x

## Implementation Steps

### 1. Create New Test Project Structure
**Location:** `src/test/resources/gradle/liberty-gradle-9-test-wrapper-app/`

**Files to create:**
- Copy all source files from existing project (`src/main/` and `src/test/`)
- Copy `build.gradle` (will be updated with LGP 4.0.0)
- Copy `Dockerfile`
- **Generate new Gradle 9.0 wrapper files** (see step 2a below)

### 2. Generate Gradle 9.0 Wrapper Files

**IMPORTANT:** The Gradle wrapper files (`gradlew`, `gradlew.bat`, `gradle-wrapper.jar`, `gradle-wrapper.properties`) must be generated using Gradle 9.0, not just copied and edited.

#### Steps to Generate Gradle 9.0 Wrapper:

1. **Navigate to the new project directory:**
   ```bash
   cd src/test/resources/gradle/liberty-gradle-9-test-wrapper-app
   ```

2. **Generate wrapper using Gradle 9.0:**
   ```bash
   gradle wrapper --gradle-version 9.0
   ```
   
   OR if you have Gradle 9.0 installed:
   ```bash
   gradle wrapper --gradle-version 9.0 --distribution-type bin
   ```

3. **This will generate/update:**
   - `gradle/wrapper/gradle-wrapper.jar` - Binary wrapper JAR for Gradle 9.0
   - `gradle/wrapper/gradle-wrapper.properties` - Points to Gradle 9.0 distribution
   - `gradlew` - Unix wrapper script (may be updated for Gradle 9.0)
   - `gradlew.bat` - Windows wrapper script (may be updated for Gradle 9.0)

**Why this is necessary:**
- The `gradle-wrapper.jar` is version-specific and contains Gradle 9.0 bootstrap code
- The wrapper scripts (`gradlew`, `gradlew.bat`) may have version-specific changes
- Simply changing the URL in `gradle-wrapper.properties` is not sufficient
- The wrapper ensures the correct Gradle version is downloaded and used

**Alternative approach if Gradle 9.0 is not installed:**
1. Copy the existing project structure
2. Update `gradle-wrapper.properties` to point to Gradle 9.0
3. Run `./gradlew wrapper --gradle-version 9.0` from the new project directory
4. This will download Gradle 9.0 and regenerate all wrapper files

After running `gradle wrapper --gradle-version 9.0`, this file will automatically contain:
```properties
distributionBase=GRADLE_USER_HOME
distributionPath=wrapper/dists
distributionUrl=https\://services.gradle.org/distributions/gradle-9.0-bin.zip

```
